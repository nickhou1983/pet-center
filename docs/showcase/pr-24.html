🔧 Workflow Repair & Prompt Enhancement

Fix feature-showcase PR creation failure + upgrade to GitHub MCP tools

What & Why

Problem: The PR feature-showcase workflow was silently failing at the "create showcase PR" step every time, despite appearing green in CI. Meanwhile, agent prompts relied on `gh` CLI commands that cannot run in the sandbox environment.

Solution: Fixed the root cause (incorrect base branch compilation) and modernized prompts to use GitHub MCP tools instead of CLI—ensuring both workflows work reliably and can run in restricted environments.

Key Capabilities

✓ Fixed PR Base Branch Routing

Corrected the compiled base branch from ${{ github.ref_name }} (which evaluates to "merge" in PR context) to hardcoded main. Now showcase PRs create successfully against the main branch.

✓ Reduced Trigger Frequency

Changed PR event triggers from [opened, synchronize, reopened] to just [opened]. Eliminates redundant showcase PR creation on every force-push or review request, keeping the showcase folder clean.

✓ GitHub MCP Tool Adoption

Upgraded agent prompts to use native GitHub `pull_requests` toolset instead of CLI. Works reliably in sandbox environments without requiring GitHub credentials—safer and more portable.

✓ Enhanced Prompt Guidance

Updated workflow documentation to explicitly instruct agents to use GitHub API tools and warn against relying on `gh` CLI in restricted environments. Improves maintainability and clarity.

Understanding the Bug & Fix

🔴 Root Cause

In GitHub Actions, during a pull_request event, github.ref_name resolves to the merge ref (e.g., 23/merge), not a real branch name. When the `create-pull-request` safe output compiled base_branch: ${{ github.ref_name }}, Git couldn't find the target branch:

Base branch: 23/merge
fatal: couldn't find remote ref 23/merge
✗ Message 1 (create_pull_request) failed: git exit code 128

🟢 Solution Applied

Added explicit base-branch: main to the safe-outputs config, ensuring showcase PRs always target the main branch regardless of the triggering PR's base:

safe-outputs:
  create-pull-request:
    + base-branch: main
    draft: true
    title-prefix: "[feature-showcase] "
